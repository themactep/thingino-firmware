# SEI Overlay Tool

Extracts embedded SEI (Supplemental Enhancement Information) JSON metadata from
[prudynt](https://github.com/themactep/thingino-firmware)-recorded MP4 files and
either burns it as on-screen display into a new video, or exports it as a
standalone ASS / SRT subtitle file.

The SEI metadata is generated by prudynt automatically whenever OSD is enabled.
It carries dynamically configured elements — timestamp, hostname, IP address,
uptime, gain, or static text — each with a type, text value, and pixel position.

```
scripts/sei-overlay.py
```

## Requirements

- **ffmpeg** with `libfreetype`, `libass`, and `libx264`
- **Python 3.8+** (stdlib only — no pip packages)

## Quick start

```bash
# Burn OSD into a new video
./scripts/sei-overlay.py video.mp4                     # → video_overlay.mp4

# Batch-process a whole directory
./scripts/sei-overlay.py ~/nfs/ing-wyze-vdb1-1564/2026/07/05/

# Export styled ASS subtitle (no re-encode)
./scripts/sei-overlay.py video.mp4 --extract-ass       # → video.ass

# Export plain-text SRT subtitle
./scripts/sei-overlay.py video.mp4 --extract-srt       # → video.srt

# Inspect the raw SEI timeline
./scripts/sei-overlay.py video.mp4 --only-extract | jq .
```

## Options

| Flag | Default | Description |
|---|---|---|
| `--font NAME` | `DejaVu Sans` | Font name for the ASS Style |
| `--border-width N` | `2` | Stroke width in pixels |
| `--codec CODEC` | `libx264` | Output video codec |
| `--crf N` | `23` | Quality (lower = bigger file) |
| `--preset NAME` | `medium` | x264 speed preset |
| `--no-rotate` | *off* | Skip the `transpose` filter; treat OSD positions as raw‑stream coordinates |
| `--dry-run` | *off* | Print the ffmpeg command without running |
| `--keep-ass` | *off* | Keep the intermediate `.ass` file after encoding |

## Output modes

### 1. Burned‑in overlay (default)

```
input.mp4  →  input_overlay.mp4
```

- Rotates the video to display orientation (respects the `rotation` field from SEI)
- Renders time‑stamp, hostname, uptime (and brightness, if present in SEI)
- Audio is stream‑copied (no re‑encode)

### 2. Standalone ASS subtitle (‑‑extract‑ass)

```
input.mp4  →  input.ass
```

Writes a styled `.ass` file with per‑element formatting:

- `\an` alignment (top‑left / top‑centre / top‑right / bottom‑left)
- White text, black outline, 32px font (fixed defaults)
- Each element gets a unique `Layer` to prevent stacking

Play it alongside the original video without re‑encoding:

```bash
ffplay -vf "subtitles=video.ass" video.mp4
```

### 3. Standalone SRT subtitle (‑‑extract‑srt)

```
input.mp4  →  input.srt
```

Plain‑text `.srt` file. All OSD elements are joined into a single subtitle line
separated by ` | `.  No styling, no positioning — maximum portability.

### 4. Raw JSON timeline (‑‑only‑extract)

```json
[
  {
    "start": 0.0,
    "end": 6.08,
    "sei": {
      "v": 1,
      "sw": 2048,
      "sh": 1536,
      "rotation": 90,
      "elements": [
        {"t": "timestamp", "text": "2026-07-05 20:40:39", "x": 10, "y": 10},
        {"t": "hostname",  "text": "ing-cam",            "x": 0,  "y": 10},
        {"t": "ipaddress", "text": "192.168.1.100",      "x": 0,  "y": 30},
        {"t": "uptime",    "text": "01:23:45",           "x": -10, "y": 10},
        {"t": "gain",      "text": "312",                "x": 10, "y": -10}
      ]
    },
    "anchor_pts": 3.0
  }
]
```

Dumps the grouped SEI segments with their anchor PTS.  Useful for debugging,
custom post‑processing, or building your own overlay pipeline.

## How it works

1. **Extract** — The raw H.264 bitstream is demuxed from the MP4 and parsed for
   SEI NAL units carrying the Thingino UUID
   (`a1b2c3d4‑e5f6‑4780‑abcd‑ef1234567890`).  Each SEI payload is mapped to a
   video frame via the MP4 sample table.

2. **Group** — Consecutive frames with identical SEI JSON are collapsed into
   time‑range segments.  The first segment is extended back to PTS 0 so OSD
   coverage starts immediately.  Each segment records an `anchor_pts` — the
   actual PTS where the SEI first appeared — used later for interpolation.

3. **Interpolate** — The `timestamp` element is interpolated at 1‑second intervals
   between SEI anchor points.  The first seconds before the initial SEI
   are extrapolated backwards (so the time‑stamp matches the filename).
   Other element types are not interpolated — they update only when the
   SEI group changes.

4. **Render** — An ASS subtitle file is generated with one `Dialogue` event
   per element per second.  The `PlayRes` is set to the actual display
   canvas size so that `fontsize` and margins are in real pixel units.
   FFmpeg's `subtitles` filter burns it in (with `transpose` if rotation ≠ 0).

## Positioning convention

The SEI `x` / `y` values follow the OSD convention from prudynt:

| Value | Meaning |
|---|---|
| `> 0` | Offset from left / top edge |
| `0` | Centred horizontally / vertically |
| `< 0` | Offset from right / bottom edge (absolute value) |

When rotation ≠ 0 the position is reinterpreted against the **display** canvas
dimensions so the text always appears at the intended screen edge regardless of
orientation.

## Batch processing

```bash
# Process every .mp4 in a directory (skips existing _overlay files)
./scripts/sei-overlay.py ~/recordings/2026/07/05/

# Custom output directory with specific quality
./scripts/sei-overlay.py ~/recordings/ --crf 28 --preset fast
```

## Known limitations

- **H.264 only.**  H.265 / HEVC SEI parsing is not yet implemented.
- **No per‑element styling.**  The SEI no longer carries font size, colours, or
  stroke width.  The overlay tool uses fixed defaults (white text, black outline,
  32px font).
